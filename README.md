# Photonic Integrated Circuit PDK Development - Research Internship Project (IIT Bhilai)

This repository contains the workspace and resources for the Silicon Photonics Process Design Kit (PDK) research internship project at the Indian Institute of Technology Bhilai (IIT Bhilai).

---

## Project Overview

The objective of this research internship is to design and build a custom Process Design Kit (PDK) using Python. Instead of drawing physical microchip layouts manually in a graphical user interface (GUI), we use code to programmatically define, parameterize, and connect photonic components (such as waveguides, bends, splitters, and couplers). The final chip designs are exported in GDSII (.gds) format, which is the industry-standard binary format sent to semiconductor foundries for fabrication.

---

## Core Concepts

### Photonic Integrated Circuits (PIC)
A Photonic Integrated Circuit is a microchip that processes and transmits data using light (photons) instead of electricity. It routes light through microscopic structures fabricated on a substrate (typically Silicon-on-Insulator, or SOI).

Key components:
* Waveguide: A microscopic channel that guides light on the chip (acting like an on-chip fiber optic cable).
* Bend: A curved waveguide structure used to route light in different directions. Euler curves are commonly used to minimize optical loss.
* Splitter (e.g., Y-junction or Multi-Mode Interferometer): Splits a single input light path into multiple outputs.
* Grating Coupler: A component that helps couple light from an external optical fiber onto the chip's waveguides.

### GDSII Layout Format
GDSII (.gds) is a hierarchical binary database file format that represents the 2D planar geometries of the integrated circuit layers. 
* Layers: Represent different materials or processing steps (e.g., silicon etching, metal contacts).
* Cells: Reusable layout blocks. A top-level cell can reference lower-level cells, allowing hierarchical design.
* Polygons and Paths: The actual geometric shapes representing the physical layout.
* Ports: Mathematical connection points on a component that specify coordinates, width, orientation angle, and waveguide type.

## Project Workspace Structure

* `demo.py`: Main introductory script that generates a waveguide and a bend, connects them, and writes the output GDS file.
* `view_gds.py`: Utility script to inspect and visualize the contents of a GDS file from the command line using Matplotlib.
* `check_libs.py`: Scanner script to verify which required and optional Python libraries are installed.
* `waveguide_bend.gds`: The GDSII output layout generated by running the demo script.
* `iPDK_Guide.md`: An in-depth reference document explaining the structure of PDKs, layer stacks, cross-sections, PCells, and routing.
* `Internship_Study_Guide.md`: A step-by-step learning guide and development procedure tailored for beginners starting from the absolute basics.
* `.venv312/`: Python 3.12 virtual environment containing the pre-installed project dependencies.

## Software Stack and Libraries

The project relies on a modern open-source Python toolchain for photonic layout design:

* Core Layout Engines:
  * gdsfactory: The primary open-source layout automation framework.
  * kfactory: Layout database management layer utilizing the KLayout C++ engine.
  * klayout: Python bindings for KLayout, used for fast GDS file input/output and geometric calculations.
  * gdstk: Fast C++ library for GDSII/OASIS creation with Python bindings.

* Mathematical and Geometric Utilities:
  * numpy: Coordinate calculation and vector mathematics.
  * scipy: Curve generation, interpolation, and mathematical optimization.
  * shapely: Planar geometry operations (boolean operations, buffering).

* Simulation and Mode Solving (Optional/Plugin libraries):
  * gplugins: Interface layer connecting gdsfactory to various simulation tools.
  * tidy3d: API client for cloud-based FDTD simulations.
  * femwell: Finite Element Method mode solver for optical waveguides.

## Getting Started

### 1. Activate the Python Virtual Environment
Open PowerShell or your terminal in Windows, navigate to the internship folder, and activate the virtual environment:

```powershell
# Navigate to the project directory
cd c:\Users\User\Downloads\Internship

# Activate the virtual environment
.\.venv312\Scripts\Activate.ps1
```

### 2. Generate the Sample Layout
Run the main script to create a waveguide and bend layout:

```powershell
python demo.py
```
This generates `waveguide_bend.gds` in the workspace directory.

### 3. Verify the Layout
You can visualize the layout using the built-in script or by opening it in the KLayout GUI application:

* To inspect via command line:
  ```powershell
  python view_gds.py waveguide_bend.gds
  ```

* To open in KLayout (if installed in the default location):
  ```powershell
  & "$env:APPDATA\KLayout\klayout_app.exe" waveguide_bend.gds
  ```
